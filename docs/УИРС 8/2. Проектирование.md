## 2.1 Функциональные требования

### Обязательные требования (реализуемые за УИРС)

- **FR-1.** Панорамирование и масштабирование холста: поддержка перемещения области просмотра (средняя кнопка мыши или правая кнопка) и масштабирование (колёсик + Ctrl), диапазон 20‑400%.
- **FR-2.** Сетка и привязка к сетке: отображение координатной сетки (шаг 24 px, настраиваемый) и snap‑to‑grid для перемещаемых и создаваемых элементов.
- **FR-3.** Добавление базовых фигур: прямоугольник, круг, ромб; добавление через кнопку панели инструментов или drag‑and‑drop из библиотеки.
- **FR-4.** Перемещение и изменение фигур: перетаскивание мышью, трансформер с 8 якорями (угловые – независимое изменение ширины и высоты, боковые – изменение только одной стороны); автоматическое центрирование текста при изменении размеров.
- **FR-5.** Создание связей между фигурами: перетаскивание от якоря исходной фигуры к якорю целевой; поддержка точек излома (добавление, перемещение, удаление).
- **FR-6.** Стилизация связей: настройка цвета, толщины линии, типа окончания (стрелка, точка, отсутствие) для начала и конца.
- **FR-7.** Выделение и удаление элементов: одиночный клик для выделения, Delete для удаления, множественное выделение прямоугольной рамкой (лассо).
- **FR-8.** История действий: отмена (Ctrl+Z) и повтор (Ctrl+Y) охватывают добавление, удаление, перемещение, изменение размеров, стили и связи.
- **FR-9.** Инспектор свойств: отображение и редактирование текста, цвета заливки/обводки, толщины обводки; для связей – тип окончания стрелки; мгновенное применение изменений.
- **FR-10.** Сохранение и загрузка диаграмм: автоматическое сохранение в Zustand‑сторе, восстановление из localStorage/IndexedDB, экспорт/импорт в JSON‑файл.
- **FR-11.** Расширяемость через JSON‑конфигурацию: динамическая загрузка описаний новых элементов без перекомпиляции; реализованы конфигурации BPMN (задача, шлюз, события), UML (класс), C4 (контейнер, компонент).
- **FR-12.** Библиотека элементов: группировка по нотациям, drag‑and‑drop на холст, автоматическое обновление при загрузке новых JSON‑конфигураций.

### Перспективные требования (дипломный этап)

- **FR-13.** Совместное редактирование в реальном времени (CRDT/Yjs, WebSocket) с отображением курсоров и выделений других пользователей.
- **FR-14.** Комментарии к элементам с панелью отображения и возможностью закрытия.
- **FR-15.** Группировка элементов для совместного перемещения, масштабирования и копирования.
- **FR-16.** Экспорт/импорт в PlantUML.
- **FR-17.** Версионирование диаграмм с откатом к предыдущим версиям.

## 2.3 Стек технологий

1. **React Konva** – высокопроизводительный 2D‑рендеринг, поддержка слоёв, трансформеров, событий мыши; декларативная интеграция через React Konva.
2. **Zustand + immer** – легковесный глобальный стор с поддержкой иммутабельных обновлений, хранит модель диаграммы и UI‑состояние.
3. **TypeScript** – статическая типизация, облегчает работу с моделью данных и снижает количество ошибок.
4. **Vite** – быстрый сборщик, обеспечивает лёгкую настройку проекта и быстрый HMR.
5. **Yjs (планируемо)** – CRDT‑библиотека для будущей совместной работы в реальном времени.
6. **Material Design Icons** – набор иконок для библиотеки элементов.

## 2.4 Архитектура редактора

Редактор построен по слоистой архитектуре **View‑Model‑Store**.

- **View (представление)** – компоненты Konva: `DiagramCanvas` (Stage), `GridLayer`, `NodeLayer`, `EdgeLayer`, `TemporaryEdge`, `TransformerLayer`, `EdgeHandlesOverlay`, `SelectionLayer`. Обрабатывают события мыши, панорамирование, масштабирование, создание связей.
- **Model (модель)** – типы данных (`DiagramDocument`, `DiagramNode`, `DiagramEdge`, `EditorState`). Реестры (`nodeRegistry`, `edgeRegistry`, `notationRegistry`) обеспечивают быстрый доступ к Konva‑объектам и описаниям нотаций.
- **Store (управление)** – Zustand‑стор (`editor.store.ts`) хранит документ, историю (undo/redo), текущий viewport, выделения и флаги интеракций. Команды (`addNode`, `updateNode`, `deleteNode`, `addEdge`, `undo`, `redo`, `setViewport` и др.) меняют состояние и автоматически создают снапшоты для истории.
- **Декларативная система нотаций** – JSON‑файлы в `public/notations/` описывают элементы, их свойства и правила связей. При инициализации приложение загружает конфиги, формирует `NodeDefinition` и регистрирует их в `notationRegistry`, после чего они становятся доступными в боковой панели.
- **Синхронизация Konva** – утилита `syncEdgeKonva.ts` обновляет `points` линий и координаты стрелок без полной перерисовки React, обеспечивая плавность.
