*В данном разделе изложены результаты системного анализа существующих решений, формулировка требований к разрабатываемому графическому редактору и обоснование выбранных нотаций.*

## 1.1 Анализ аналогов

Для обоснования проектных решений и выявления требований к разрабатываемому графическому редактору был проведён анализ современных веб-ориентированных инструментов для создания диаграмм и нотаций. Основное внимание уделялось следующим аспектам: поддержка необходимых типов диаграмм (C4, BPMN, UML, ER, DFD), возможности совместной работы, расширяемость, производительность и интеграционный потенциал.

### 1.1.1 Обзор аналогов

**Draw.io** – один из самых распространённых бесплатных редакторов диаграмм с открытым исходным кодом. Поддерживает множество нотаций: UML, BPMN, ER, архитектурные схемы, C4 (через плагины). Интегрируется с Google Drive, OneDrive, GitHub.
**Сильные стороны:** широкая библиотека элементов, экспорт в различные форматы (PNG, SVG, XML), работа с пользовательскими шаблонами.
**Слабые стороны:** отсутствие встроенной совместной работы в реальном времени, ограниченная кастомизация нотаций без доработки исходного кода.

**Miro** – онлайн-доска для совместной работы, позволяющая создавать диаграммы, майнд-карты, пользовательские истории. Популярен в Agile‑командах.
**Сильные стороны:** отличная поддержка коллаборации (курсоры, комментарии, видеозвонки), удобное перетаскивание элементов, богатая библиотека шаблонов.
**Слабые стороны:** слабая поддержка специфических нотаций (например, C4 или BPMN 2.0), отсутствие экспорта в форматы, понятные для моделирования (PlantUML, XML), проприетарность и ограничения бесплатной версии.

**Lucidchart** – коммерческий инструмент для создания диаграмм с упором на корпоративное использование. Поддерживает UML, BPMN, ER, архитектурные диаграммы, а также интеграцию с Atlassian, Salesforce.
**Сильные стороны:** хорошая производительность при работе с большими схемами, возможность создания собственных библиотек элементов, версионность.
**Слабые стороны:** высокая стоимость, нет открытого API для расширения нотаций, совместная работа требует постоянного подключения к интернету.

**React Flow** – библиотека с открытым исходным кодом для построения графовых диаграмм на React. Используется для создания кастомных редакторов (оркестрация данных, машинное обучение).
**Сильные стороны:** отличная производительность, поддержка кастомных узлов и связей, модульная архитектура, возможность интеграции с Redux/Zustand.
**Слабые стороны:** изначально не предназначена для бизнес‑нотаций (BPMN, C4) – требуется значительная доработка, отсутствие готового UI для панелей инструментов и инспектора.

**Excalidraw** – минималистичный редактор для создания эскизов и диаграмм от руки.
**Сильные стороны:** простота использования, открытый исходный код, хорошая поддержка коллаборации (WebSocket + CRDT).
**Слабые стороны:** не подходит для строгих нотаций (BPMN, UML), нет системы типов элементов, сложно расширять.

### 1.1.3 Выводы по анализу

Проведённый анализ показывает, что ни один из существующих инструментов не удовлетворяет в полной мере требованиям проекта:

1. Draw.io и Lucidchart имеют богатую библиотеку нотаций, но не позволяют легко добавлять новые типы элементов без модификации исходного кода, а также не предоставляют встроенных механизмов совместной работы.
2. Miro хорош для коллаборации, но не подходит для формальных нотаций (C4, BPMN, UML) и не позволяет экспортировать диаграммы в специализированные форматы (PlantUML).
3. React Flow даёт гибкость и производительность, однако требует значительной доработки для поддержки бизнес‑нотаций, а также не включает готовых UI‑компонентов (библиотека элементов, инспектор свойств).
4. Excalidraw демонстрирует современный подход к коллаборации (CRDT), но не пригоден для строгого моделирования.

Таким образом, целесообразна разработка собственного графического редактора, который сочетал бы:
- Расширяемую архитектуру (возможность добавления новых нотаций через JSON‑конфигурацию);
- Высокую производительность (на базе Konva и прямого управления DOM‑слоями);
- Поддержку совместной работы (с заделом на внедрение CRDT);
- Бесшовную интеграцию с уже разработанным текстовым редактором и файловой системой.
