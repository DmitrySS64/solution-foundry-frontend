В ходе проектирования была сформирована структура клиентской части веб-приложения для ведения проектной документации. На основании аналитического раздела определены пользовательские роли, варианты использования, функциональные и нефункциональные требования. Эти материалы задали основу для проектирования маршрутов, пользовательских потоков, моделей данных и интерфейсной структуры.

Для реализации приложения выбран стек React, TypeScript, Vite, TanStack Router, Zustand, Lexical, Konva, i18next и Yjs. Выбор указанных технологий обусловлен требованиями к реактивному интерфейсу, типобезопасности, расширяемости редакторов, высокой скорости разработки и совместному редактированию документов в реальном времени.

Архитектура клиентской части спроектирована с применением Feature-Sliced Design. Такой подход позволяет разделять код по слоям и предметным областям, сохраняя понятные границы между страницами, виджетами, функциональными модулями, сущностями и общими компонентами. Для графического редактора дополнительно описана модель данных, включающая документы, узлы, связи, стили, нотации и служебное состояние интерфейса.

В результате проектирования сформирована основа для реализации приложения. Проектные решения обеспечивают расширение функциональности, подключение серверной части, развитие ролевой модели и синхронизацию документов в реальном времени.
