Модель данных клиентской части описывает основные сущности, с которыми пользователь взаимодействует при работе с проектной документацией. Наибольшее значение имеют данные графического редактора, поскольку они содержат пространственные координаты, свойства нотаций, связи между элементами и параметры отображения холста. Текстовые документы обрабатываются редактором Lexical, который использует собственное внутреннее представление состояния редактора и сериализуется в JSON для хранения и синхронизации.

Центральной сущностью графического редактора является документ диаграммы. Он включает набор узлов и набор связей. Такая структура удобна для сериализации, сохранения в локальном хранилище и последующей передачи на серверную часть.

```ts
interface DiagramDocument {
    nodes: DiagramNode[]
    edges: DiagramEdge[]
}
```

Узел диаграммы описывает визуальный элемент нотации. В его составе хранятся идентификатор, тип, координаты, размеры, угол поворота, текстовая метка, стили оформления, сведения о нотации и список связанных ребер. Наличие поля `notation` позволяет связывать конкретный элемент холста с декларативным описанием из JSON-конфигурации BPMN, UML или C4.

```ts
interface DiagramNode {
    id: string
    type: string
    x: number
    y: number
    width: number
    height: number
    rotation: number
    label: string
    style: NodeStyle
    textStyle: NodeTextStyle
    notation?: NodeNotation
    edges: {
        id: string
        direction: 'in' | 'out'
    }[]
    createdAt: number
    updatedAt: number
}
```

Связь между элементами представлена отдельной сущностью. Она хранит начальную и конечную точки, привязку к узлам, тип линии, управляющие точки излома, стиль отображения и необязательную подпись. Разделение узлов и связей упрощает перерасчет геометрии при перемещении элементов и позволяет изменять линию независимо от связанных фигур.

```ts
interface DiagramEdge {
    id: string
    source: EdgeAnchor
    target: EdgeAnchor
    type: 'straight' | 'orthogonal' | 'bezier'
    controlPoints: EdgePoint[]
    style: {
        stroke: string
        strokeWidth: number
        dash?: number[]
        startCap?: 'none' | 'arrow'
        endCap?: 'none' | 'arrow'
    }
    label?: string
}
```

Для поддержки расширяемости нотаций используется отдельная модель `NodeNotation`. Она содержит название элемента, набор редактируемых свойств и примитивы, на основании которых формируется изображение узла. Благодаря этому добавление новых типов элементов не требует изменения базовой модели диаграммы: достаточно описать тип, размеры, свойства и набор графических примитивов.

Состояние редактора дополнительно включает историю изменений, текущее выделение, параметры области просмотра и признаки пользовательского взаимодействия. История хранит прошлые и будущие снимки документа, что обеспечивает работу команд отмены и повтора. Выделение содержит идентификаторы выбранных объектов, а область просмотра задает смещение и масштаб холста.

Таким образом, модель данных разделяет предметные сущности документа и служебное состояние интерфейса. Это упрощает сохранение документов, снижает связанность между компонентами редактора и создает основу для синхронизации с серверной частью.
