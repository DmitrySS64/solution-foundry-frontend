Выпускная работа посвящена проектированию и реализации клиентской части веб-приложения для совместного ведения проектной документации. В состав приложения входят средства навигации по проектам, текстовый редактор, графический редактор нотаций, механизм локализации пользовательского интерфейса и совместное редактирование документов в реальном времени.

Ключевые слова: веб-приложение, клиентская часть, React, TypeScript, Feature-Sliced Design, TanStack Router, Zustand, Lexical, Konva, i18next, проектная документация, графические нотации, совместная работа.

Объектом исследования являются методы и технологии, применяемые при разработке клиентской части современных веб-приложений. Предмет исследования составляет архитектура реактивного одностраничного приложения, выполняющего роль тонкого клиента для работы с текстовыми и графическими артефактами проектной документации.

Цель работы заключается в проектировании и реализации клиентской части веб-приложения, ориентированного на создание, редактирование и сопровождение проектной документации в единой пользовательской среде.

В ходе выполнения работы проведен аналитический обзор существующих систем управления документацией и графического моделирования, сформулированы функциональные и нефункциональные требования, определена ролевая модель, выполнено сравнение технологических решений и спроектирована архитектура клиентской части. На основе проведенного анализа реализовано приложение с использованием React и TypeScript.

Основные характеристики разработанного решения: язык программирования TypeScript; библиотека React для построения компонентного интерфейса; TanStack Router для типизированной маршрутизации; Zustand и Immer для управления состоянием; Lexical для реализации блочного текстового редактора; Konva и React Konva для построения графического редактора; i18next и react-i18next для локализации интерфейса. Область применения работы охватывает веб-приложения, предназначенные для сопровождения проектной, аналитической и архитектурной документации.

Практическая значимость результата состоит в объединении текстового и графического редактирования в рамках одного интерфейса. Реализованная синхронизация на базе CRDT снижает зависимость команды от разрозненных инструментов, поддерживает одновременную работу нескольких пользователей и обеспечивает согласованное состояние документов при параллельных изменениях.
