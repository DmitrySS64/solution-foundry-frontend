В приложении приведены ключевые фрагменты клиентской части, иллюстрирующие основные проектные решения. Полные исходные файлы размещены в репозитории проекта.

Листинг А.1 - создание маршрутизатора приложения

```ts
const router = createRouter({
    routeTree,
    defaultPreload: 'intent',
    scrollRestoration: true,
    defaultStructuralSharing: true,
    defaultPreloadStaleTime: 0,
});
```

Данный фрагмент демонстрирует настройку TanStack Router. Параметр `defaultPreload` включает предварительную загрузку маршрута при намерении пользователя перейти на страницу, а `scrollRestoration` восстанавливает положение прокрутки после навигации.

Листинг А.2 - структура глобального состояния графического редактора

```ts
interface EditorState {
    document: DiagramDocument

        past: DiagramDocument[]
        future: DiagramDocument[]
    }
    selection: {
        ids: string[]
    }
    viewport: {
        x: number

```

В состоянии редактора разделяются содержимое диаграммы, история изменений, выделение и параметры области просмотра. Такое разделение упрощает обработку действий пользователя и сериализацию документа.

i18n

Листинг А.4 - регистрация декларативного элемента нотации

```ts
class ElementRegistry {
    private definitions = new Map<string, IElementDefinition>();

    register<T extends IBaseElement>(definition: IElementDefinition<T>): void {
        this.definitions.set(definition.type, definition);
    }

    createElement(type: string, position: { x: number; y: number }) {
        const definition = this.definitions.get(type);
        return definition ? definition.create(position) : null;
    }
}
```

Реестр элементов отделяет описание нотации от конкретных компонентов холста. Благодаря этому новые типы элементов могут подключаться через регистрацию определения.
