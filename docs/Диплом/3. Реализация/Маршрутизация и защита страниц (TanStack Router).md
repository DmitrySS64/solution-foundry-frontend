Маршрутизация реализована с использованием библиотеки TanStack Router, обеспечивающей типизацию параметров маршрутов, вложенные макеты и ленивую загрузку страниц. Такой подход соответствует архитектуре одностраничного приложения: переход между разделами выполняется на стороне клиента, а общие элементы интерфейса не пересоздаются при каждом изменении адреса.

Структура маршрутов
```
src/app/routes/
├── __root.tsx          # Корневой маршрут (layout)
├── index.tsx           # Главная страница
├── auth/
│   ├── login.tsx       # Вход
│   └── register.tsx    # Регистрация
├── projects.tsx        # Список проектов
├── project/
│   └── $id.tsx         # Страница проекта (динамический параметр)
└── document/
    └── $id.edit.tsx    # Редактирование документа
```

В фактической реализации маршруты создаются программно в модуле `src/app/routes/index.tsx`. Корневой маршрут формирует базовую точку входа, после чего подключается маршрут с верхней панелью приложения. Внутри него размещается маршрут с боковой навигацией, используемый для рабочих страниц: списка последних изменений, страницы проекта, текстового редактора и графического редактора.

Для страниц, которые не требуются при первом открытии приложения, применяется `lazyRouteComponent`. Благодаря этому код страницы проекта, Markdown-редактора и редактора диаграмм загружается только при переходе на соответствующий адрес. Данное решение уменьшает начальный объем загружаемого JavaScript и особенно важно для редакторов, поскольку они подключают дополнительные библиотеки и собственные компоненты управления.

Динамические маршруты используются для страниц, связанных с конкретными сущностями. Например, путь `/project/${id}` открывает проект с заданным идентификатором, а маршруты `/document/${id}` и `/diagram/${id}` позволяют обращаться к отдельному текстовому или графическому документу. Типизация параметров маршрута снижает риск ошибки при обращении к идентификатору внутри компонента страницы.

Защита страниц на проектном уровне реализована через проверки перед загрузкой маршрута. Для TanStack Router таким механизмом выступает `beforeLoad`, где выполняется проверка наличия пользователя, его роли и прав доступа к проекту. Маршрутизируемая структура приложения разделяет публичные и рабочие экраны, а проверка учетных данных связывает навигацию с серверной сессией пользователя.

Параметры маршрутизатора включают предварительную загрузку по намерению пользователя, восстановление позиции прокрутки и структурное переиспользование данных. Это повышает отзывчивость интерфейса и делает навигацию между страницами ближе к поведению настольного приложения.
