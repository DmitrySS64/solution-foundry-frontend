Навигация по проектам реализуется как один из основных пользовательских сценариев приложения. Пользователь должен иметь возможность открыть список проектов, перейти к конкретному проекту, просмотреть его структуру и выбрать документ для редактирования. В клиентской части этот сценарий связывает маршрутизацию, боковую панель, страницу проекта и редакторы документов.

Страница проекта размещается по динамическому маршруту `/project/${id}`. Идентификатор проекта передается через параметры маршрута, что позволяет открывать разные проекты одним и тем же компонентом страницы. Внутри страницы отображаются элементы проектной структуры: разделы, файлы, действия над ними и переходы к редакторам.

Файловая структура представлена на уровне интерфейса и связана с проектной моделью данных. Для каждого элемента отображаются название, тип, визуальная иконка и доступные действия. Текстовые документы направляют пользователя на маршрут `/document/${id}`, а диаграммы - на маршрут `/diagram/${id}`. Благодаря такому разделению каждый тип артефакта открывается в собственном редакторе, сохраняя общий контекст приложения.

Боковая навигация выполняет роль постоянного ориентира. Она помогает пользователю быстро вернуться к списку последних изменений, перейти на страницу проекта или открыть другой раздел. При проектировании учитывалось, что работа с документацией часто требует многократного переключения между файлами, поэтому навигационные элементы должны оставаться доступными на рабочих страницах.

Файловая система связана с серверной моделью проектов. Через API выполняются загрузка дерева файлов, создание и удаление документов, переименование, перемещение элементов и управление правами доступа. Структура маршрутов и компонентов сохраняет единый пользовательский поток при переходе между документами разных типов.
