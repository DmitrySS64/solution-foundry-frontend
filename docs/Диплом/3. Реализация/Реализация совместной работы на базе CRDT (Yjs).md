Совместная работа на базе CRDT реализована с использованием библиотеки Yjs. Она предоставляет структуры совместного документа, поддерживает синхронизацию через WebSocket и используется вместе с редакторами, применяемыми в проекте.

Для текстового редактора выполнена интеграция через пакет `@lexical/yjs`. Состояние Lexical связывается с Yjs-документом, а изменения текста, структуры блоков и вложенных элементов распространяются между участниками сессии. Пользователь видит результат своего действия сразу, а удаленные изменения применяются после получения соответствующих CRDT-обновлений.

Для графического редактора совместная модель представлена через Yjs-коллекции: карту узлов, карту связей и объект параметров холста. Каждый узел и каждая связь имеют устойчивый идентификатор, поэтому изменение координат, размеров, подписи или стиля передается как операция над конкретной сущностью. Такой подход хорошо согласуется с реализованной моделью `DiagramDocument`.

Отдельной частью совместной работы является присутствие пользователей. Сведения о курсорах, выделенных элементах, имени участника и его цвете передаются через awareness-механизм Yjs. Эти данные не относятся к содержимому документа и не требуют долговременного хранения, однако они повышают прозрачность совместного редактирования.

CRDT-подход позволяет разрешать типичные конфликтные ситуации без ручного вмешательства. Например, два пользователя могут одновременно изменить разные свойства одного узла; после обмена обновлениями документ приводится к согласованному состоянию. При одновременном редактировании одного текстового блока порядок операций определяется механизмами Yjs, что снижает риск потери данных.

Сетевое взаимодействие выполняется через WebSocket-провайдер. Он связывает участников сессии, передает CRDT-обновления, восстанавливает состояние документа при подключении нового пользователя и использует механизм авторизации для ограничения доступа к совместному редактированию.
