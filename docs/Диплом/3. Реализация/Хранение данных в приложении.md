В приложении используется несколько уровней хранения данных: локальное состояние компонентов, глобальное состояние редактора и персистентное хранение в браузере. Такое разделение позволяет не смешивать кратковременные параметры интерфейса с данными документа, которые должны сохраняться между пользовательскими сессиями.

Локальное состояние применяется для небольших элементов интерфейса: открытого меню, состояния формы, временного значения поля ввода или признака отображения модального окна. Эти данные принадлежат конкретному компоненту и не требуют передачи в другие части приложения.

Глобальное состояние графического редактора реализовано с использованием Zustand и Immer. В хранилище находятся документ диаграммы, история изменений, текущее выделение, параметры области просмотра и признаки взаимодействия с холстом. Immer позволяет изменять вложенные структуры в императивном стиле, сохраняя при этом иммутабельную модель обновления состояния.

Состояние редактора включает следующие группы данных:
- `document` - массив узлов и массив связей диаграммы;
- `history` - снимки документа для операций отмены и повтора;
- `selection` - идентификаторы выбранных элементов;
- `viewport` - координаты смещения и масштаб холста;
- `interaction` - признаки перетаскивания, наведения и активного редактирования связи.

Персистентное хранение используется для сохранения документов диаграмм, пользовательских настроек и локальных синхронизационных данных. Данные диаграммы сериализуются в JSON и сохраняются в браузерном хранилище. При повторном открытии страницы документ восстанавливается из сохраненного состояния, что позволяет продолжить работу без потери контекста.

Для режима совместной работы применяется IndexedDB совместно с механизмами Yjs persistence provider. Такой подход хранит локальные CRDT-обновления, поддерживает работу при нестабильном соединении и синхронизирует накопленные изменения после восстановления связи.

Выбранная организация хранения данных разделяет ответственность между уровнями приложения. Локальные данные интерфейса остаются в компонентах, состояние редактора централизовано в Zustand, долговременное хранение вынесено в браузерные механизмы, а согласование изменений между пользователями выполняется через Yjs и серверный WebSocket-провайдер.
