Интернационализация пользовательского интерфейса реализована с использованием библиотек i18next и react-i18next. Данный инструмент выбран из-за развитой экосистемы, поддержки React-компонентов и возможности загружать переводы из внешних JSON-файлов.

Конфигурация локализации расположена в модуле `src/shared/i18n/config`. При инициализации подключаются `i18next-http-backend`, `i18next-browser-languagedetector` и адаптер `initReactI18next`. Первый модуль отвечает за загрузку файлов переводов из каталога `public/locales`, второй определяет язык пользователя, а адаптер связывает экземпляр i18next с React-приложением.

В приложении заданы русский и английский языки. Русский язык используется как основной, английский указан в качестве резервного. Такой вариант позволяет отображать интерфейс даже при отсутствии части переводов в выбранной локали.

Тексты интерфейса вынесены из компонентов в JSON-ресурсы. Компоненты обращаются к ним через хук `useTranslation`, а для фрагментов с вложенной разметкой применяется компонент `Trans`. Это снижает связанность интерфейсного кода с конкретным языком и упрощает последующее добавление новых локалей.

Отдельные пространства переводов применяются для общих текстов и для графического редактора. Такое разделение удобно при росте приложения: переводы редактора могут развиваться независимо от переводов навигации, страниц проекта и служебных элементов интерфейса.

Использование i18next также повышает сопровождаемость приложения. При изменении формулировок достаточно отредактировать ресурс перевода, не затрагивая компонентную структуру и бизнес-логику. Для приложения, ориентированного на командную работу и потенциальное использование разными группами пользователей, поддержка локализации является существенным нефункциональным требованием.
